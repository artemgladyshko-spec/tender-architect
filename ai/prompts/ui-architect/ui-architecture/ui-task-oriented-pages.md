# Task-Oriented Page Design

Many applications consist of simple task workflows.

Example:

upload document  
run analysis  
view result

Pages should reflect this workflow clearly.

---

## Task Page Structure

Common structure:

PageTitle  
UserInput  
PrimaryAction  
ResultsSection

Example:

Tender Analyzer

File Upload  
Run Analysis  
Analysis Results

---

## Input Design

User inputs must be clear and simple.

Examples:

file upload  
text input  
dropdown selection

Avoid overwhelming the user with many inputs.

---

## Action Flow

Primary actions should be obvious.

Example:

Run Analysis

Users should understand the next step immediately.

---

## Result Presentation

Results should appear in structured sections.

Examples:

summary  
extracted data  
recommendations

Use clear blocks or containers.

---

## Empty State

Before processing, result areas should show a placeholder.

Example:

"No results yet."

This helps guide user expectations.
