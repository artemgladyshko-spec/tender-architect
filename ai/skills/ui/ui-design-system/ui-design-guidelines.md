# UI Design Guidelines

Interfaces must prioritize clarity, simplicity, and usability.

## Design Philosophy

Use **minimal design**.

The UI should feel:

clean  
focused  
professional

Avoid unnecessary visual elements.

---

## Color Usage

Use neutral color palettes.

Examples:

light gray backgrounds  
dark readable text  
one accent color for primary actions

Avoid bright or distracting color combinations.

---

## Typography

Use clean readable typography.

Rules:

- clear hierarchy
- large page titles
- readable body text

Avoid too many font sizes.

---

## Containers

Use simple containers for grouping UI.

Examples:

cards  
panels  
sections

Containers should provide spacing and structure.

Avoid deep nesting of containers.

---

## Action Design

Primary actions must be clearly visible.

Example:

Run Analysis

Secondary actions should appear visually lighter.

Example:

Upload File

Avoid placing too many buttons together.

---

## Visual Hierarchy

Important information should appear first.

Example page order:

Title  
User input  
Primary action  
Results
