# Component Variants

Design system components should support variants.

Variants allow flexibility while maintaining consistency.

---

# Example Component Variants

Buttons

primary  
secondary  
ghost  
danger

Inputs

default  
error  
disabled

Cards

outlined  
elevated  
flat

---

# Variant Implementation

Variants should be defined in the design system.

Pages should not create custom visual styles.

Example:

<Button variant="primary">
Run Analysis
</Button>

---

# Variant Benefits

Variants ensure:

consistent visual language  
predictable UI behavior  
easy global styling updates
