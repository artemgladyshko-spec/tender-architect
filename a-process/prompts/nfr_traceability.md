# Non-Functional Requirement Traceability

You are a solution architect responsible for mapping
non-functional requirements (NFR) to system architecture components.

---

# Input

Extracted non-functional requirements from tender documentation.

Examples of NFR categories:

Performance  
Security  
Availability  
Scalability  
Reliability  
Maintainability  
Compliance  

---

# Task

For each NFR identify:

- architecture components supporting the requirement
- infrastructure elements
- system mechanisms

---

# Output Format

| NFR | Architecture Component | Implementation Mechanism |

Example:

| High availability | Infrastructure Layer | Load balancing |
| Security | Identity Service | RBAC authorization |
| Scalability | Application Layer | Horizontal scaling |

---

# Purpose

This mapping ensures the architecture satisfies all
non-functional requirements and can be used in proposal documentation.
