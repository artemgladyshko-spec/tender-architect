Tender Generation Workflow

Step 1 — Analyze tender documentation

Inputs:
- ToR
- annexes
- contracts
- deadlines
- NFR

Outputs:
a-process/analysis

Extract:

- functional requirements
- non functional requirements
- constraints
- integrations
- deadlines
- deliverables


Step 2 — Generate PBS

Input:
requirements analysis

Output:
a-process/pbs

Generate:

- subsystem list
- dependency structure


Step 3 — Generate domain model

Input:
PBS + UI prototype

Output:
a-process/domain_model

Generate:

- entities
- actors
- user flows


Step 4 — Generate architecture

Output:
a-process/architecture

Generate:

- system architecture
- frontend architecture
- backend architecture
- security architecture


Step 5 — Generate database design

Output:
a-process/database


Step 6 — Generate API design

Output:
a-process/api


Step 7 — Generate estimation

Output:
a-process/estimation


Step 8 — Generate final proposal

Output:
